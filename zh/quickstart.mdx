本页会带你从一个空环境走到一个模型驱动的最小 coding agent，再到可视化 trace，全程控制在 2 分钟左右。

这就是 QitOS runtime 实际运行时的体验：

![QitOS runtime 快照](/images/qitos-cli.png)

<Note>
如果你还没有远程模型服务和 API Key，建议先看 [前置准备](/zh/prerequisites)。里面说明了去哪里注册、如何创建 API Key，以及如何先验证 OpenAI-compatible provider 是否可用。
</Note>

## 前置条件

- Python 3.10 或更高版本
- `pip`
- 一个 OpenAI-compatible API key

## 操作步骤

<Steps>
  <Step title="安装 QitOS">
    ```bash
    pip install "qitos[models]"
    ```
  </Step>

  <Step title="设置 provider 配置">
    ```bash
    export OPENAI_API_KEY="sk-..."
    export OPENAI_BASE_URL="https://api.siliconflow.cn/v1/"
    export QITOS_MODEL="Qwen/Qwen3-8B"
    ```

    <Tip>
      任意 OpenAI-compatible endpoint 都可以使用。如果你不是直接使用 OpenAI，请把 `OPENAI_BASE_URL` 指向对应 provider。
    </Tip>
  </Step>

  <Step title="运行最小 coding agent">
    ```bash
    qit demo minimal
    ```

    这个 demo 会先种一个最小 bug workspace，配置真实模型，让最小 coding agent 修复 bug、执行验证，并写出 qita 可识别的 trace。你会看到类似输出：

    ```
    model_name: Qwen/Qwen3-8B
    workspace: .../playground/minimal_coding_agent
    target_file: buggy_module.py
    trace_run: .../runs/qitos_minimal_coding_<timestamp>
    final_result: Patch applied and verification passed.
    stop_reason: final
    next_step: qita board --logdir runs
    ```
  </Step>

  <Step title="用 qita 查看运行轨迹">
    ```bash
    qita board --logdir runs
    ```

    在浏览器中打开 [http://127.0.0.1:8765](http://127.0.0.1:8765)。board 会列出 logdir 中的所有 run，包括 step 数、stop reason 和 final result。点击任意 run 的 **view** 查看逐步轨迹，或用 **replay** 做回放。

    <Tip>
      打包好的 demo 与 [`examples/quickstart/minimal_agent.py`](https://github.com/Qitor/qitos/blob/main/examples/quickstart/minimal_agent.py) 讲的是同一条最小 coding-agent 路径。想读代码或改它时，就从这个例子文件开始。
    </Tip>
  </Step>
</Steps>
