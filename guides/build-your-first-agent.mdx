Every QitOS agent follows one execution path: `AgentModule + Engine + Trace`. You define the strategy; the Engine owns the loop. This guide walks through every piece using the minimal calculator agent from `examples/quickstart/minimal_agent.py`.

## Prerequisites

- Python 3.10+
- An API key for any OpenAI-compatible endpoint (optional for the minimal example — no LLM is needed)

<Steps>

<Step title="Install qitos">

Install the package in development mode from the repo root, or from PyPI:

```bash
pip install qitos
# for model support and YAML config
pip install "qitos[models,yaml]"
```

If you plan to connect to an LLM, export your key:

```bash
export OPENAI_API_KEY="sk-..."
# or for SiliconFlow / other providers
export QITOS_API_KEY="sf-..."
export OPENAI_BASE_URL="https://api.siliconflow.cn/v1/"
```

</Step>

<Step title="Define your state">

Every agent carries a typed state object that persists across all steps. Subclass `StateSchema` and add whatever fields your agent needs:

```python
from dataclasses import dataclass
from qitos import StateSchema

@dataclass
class CalcState(StateSchema):
    pass  # base class provides: task, current_step, max_steps, final_result, stop_reason
```

`StateSchema` always provides these fields — you never need to redeclare them:

| Field | Type | Purpose |
|---|---|---|
| `task` | `str` | The objective text passed to `run()` |
| `current_step` | `int` | Step counter incremented by the Engine |
| `max_steps` | `int` | Budget ceiling |
| `final_result` | `str \| None` | Set this to stop the run with a result |
| `stop_reason` | `str \| None` | Written by the Engine on exit |

<Tip>
Add domain-specific fields — scratchpads, partial plans, fetched content — as regular dataclass fields. Keep them serializable; the Engine traces the state at every step.
</Tip>

</Step>

<Step title="Implement init_state and reduce">

`init_state` creates the starting state for a run. `reduce` folds the result of each action back into state:

```python
from typing import Any
from qitos import Action, AgentModule, Decision, StateSchema

@dataclass
class CalcState(StateSchema):
    pass

class MinimalAgent(AgentModule[CalcState, dict[str, Any], Action]):

    def init_state(self, task: str, **kwargs: Any) -> CalcState:
        return CalcState(task=task, max_steps=int(kwargs.get("max_steps", 2)))

    def reduce(
        self,
        state: CalcState,
        observation: dict[str, Any],
        decision: Decision[Action],
    ) -> CalcState:
        action_results = observation.get("action_results", [])
        if action_results:
            state.final_result = str(action_results[0])
        return state  # always return state
```

<Warning>
You must return `state` from `reduce`. Forgetting to return it is the most common mistake — the Engine will raise a validation error.
</Warning>

</Step>

<Step title="Add tools">

Use the `@tool` decorator and register with `ToolRegistry`:

```python
from qitos import Action, AgentModule, Decision, StateSchema, ToolRegistry, tool

class MinimalAgent(AgentModule[CalcState, dict[str, Any], Action]):
    def __init__(self):
        registry = ToolRegistry()

        @tool(name="add")
        def add(a: int, b: int) -> int:
            return a + b

        registry.register(add)
        super().__init__(tool_registry=registry)
```

For the minimal example there is no LLM — the agent overrides `decide()` to return a hardcoded `Decision`:

```python
    def decide(
        self,
        state: CalcState,
        observation: dict[str, Any],
    ) -> Decision[Action]:
        if state.current_step == 0:
            return Decision.act(
                [Action(name="add", args={"a": 40, "b": 2})],
                rationale="compute_answer",
            )
        return Decision.final("42")
```

`Decision` has four canonical modes:

| Mode | Factory | When to use |
|---|---|---|
| `act` | `Decision.act(actions, rationale)` | Execute one or more tools |
| `final` | `Decision.final(answer)` | Produce a result and stop |
| `wait` | `Decision.wait(rationale)` | Skip tool execution this step |
| `branch` | `Decision.branch(candidates)` | Tree-of-Thought branching |

</Step>

<Step title="Run the agent">

Call `agent.run()` with `task`, `workspace`, and `max_steps`:

```python
from pathlib import Path

WORKSPACE = Path("./playground/minimal_agent")

def main() -> None:
    WORKSPACE.mkdir(parents=True, exist_ok=True)
    agent = MinimalAgent()
    result = agent.run(
        task="compute 40 + 2",
        workspace=str(WORKSPACE),
        max_steps=2,
        return_state=True,
    )
    print("final_result:", result.state.final_result)
    print("stop_reason:", result.state.stop_reason)
```

When `return_state=True`, `run()` returns an `EngineResult` with:
- `result.state` — final typed state
- `result.records` — per-step `StepRecord` list
- `result.step_count` — number of steps taken

Without `return_state=True`, `run()` returns `result.state.final_result` directly.

<Tip>
Trace artifacts are written to `./runs/<run_id>/` automatically. Pass `trace_logdir="./my_runs"` to change the output directory.
</Tip>

</Step>

<Step title="Inspect with qita board">

After the run completes, open the web board to inspect the trace:

```bash
qita board --logdir ./runs
```

The board starts on `http://127.0.0.1:8765`. Use the run cards to open the trace view, step navigator, and timeline.

```bash
# change host/port
qita board --logdir ./runs --host 0.0.0.0 --port 9000
```

For a single run:

```bash
qita replay --run <run_dir>                   # step-by-step playback
qita export --run <run_dir> --html out.html  # standalone HTML
```

</Step>

</Steps>

## Complete example

Here is the full `examples/quickstart/minimal_agent.py`:

```python
"""Quickstart: smallest runnable AgentModule + Engine flow."""

from __future__ import annotations

from dataclasses import dataclass
from pathlib import Path
from typing import Any

from qitos import Action, AgentModule, Decision, StateSchema, ToolRegistry, tool

TASK = "compute 40 + 2"
WORKSPACE = Path("./playground/minimal_agent")
MAX_STEPS = 2


@dataclass
class CalcState(StateSchema):
    pass


class MinimalAgent(AgentModule[CalcState, dict[str, Any], Action]):
    def __init__(self):
        registry = ToolRegistry()

        @tool(name="add")
        def add(a: int, b: int) -> int:
            return a + b

        registry.register(add)
        super().__init__(tool_registry=registry)

    def init_state(self, task: str, **kwargs: Any) -> CalcState:
        return CalcState(task=task, max_steps=int(kwargs.get("max_steps", MAX_STEPS)))

    def decide(self, state: CalcState, observation: dict[str, Any]) -> Decision[Action]:
        if state.current_step == 0:
            return Decision.act([Action(name="add", args={"a": 40, "b": 2})], rationale="compute_answer")
        return Decision.final("42")

    def reduce(self, state: CalcState, observation: dict[str, Any], decision: Decision[Action]) -> CalcState:
        action_results = observation.get("action_results", []) if isinstance(observation, dict) else []
        if action_results:
            state.final_result = str(action_results[0])
        return state


def main() -> None:
    WORKSPACE.mkdir(parents=True, exist_ok=True)
    agent = MinimalAgent()
    result = agent.run(
        task=TASK,
        workspace=str(WORKSPACE),
        max_steps=MAX_STEPS,
        return_state=True,
    )
    print("workspace:", WORKSPACE)
    print("final_result:", result.state.final_result)
    print("stop_reason:", result.state.stop_reason)


if __name__ == "__main__":
    main()
```

## Common mistakes

| Mistake | Symptom | Fix |
|---|---|---|
| Forget `return state` in `reduce` | `StateValidationError` or `None` state | Always end `reduce` with `return state` |
| Never set `state.final_result` | Run hits `max_steps` and exits with `stop_reason=budget_steps` | Set `state.final_result = "..."` when done, or return `Decision.final(answer)` |
| Wrong prompt-parser pairing | Parse errors every step | Match your system prompt format to the parser — e.g., `REACT_SYSTEM_PROMPT` → `ReActTextParser` |
| `max_steps` not passed to `init_state` | Agent always uses the default budget | Forward `kwargs.get("max_steps", DEFAULT)` in `init_state` |
| Tool not registered before `super().__init__()` | `KeyError` on tool lookup | Build your `ToolRegistry` first, then call `super().__init__(tool_registry=registry)` |
